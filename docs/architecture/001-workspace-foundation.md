# Workspace Foundation

## Purpose

This document defines the initial Eraxis repository structure.

## Architecture

Eraxis follows a modular platform architecture.

Hierarchy:

Core → Plugin → Module → Feature → Component

## Current Scope

The repository foundation includes:

* apps
* packages
* plugins
* documentation

## Excluded

The following are intentionally deferred:

* Authentication
* Database
* Supabase
* HRMS
* Fleet
* Inventory
* CRM

These will be implemented incrementally after the platform foundation is stable.
